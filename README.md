# Yuri Rewrite

Yuri Rewrite 是一个 Windows 优先的本地桌面工具，用于导入 TXT 小说、调用用户自定义 AI 模型分析文本，并将小说改写成双女主 / 百合版本。

应用采用本地优先设计：小说原文、章节拆分、批次文本、分析结果、改写稿、一致性资产、模型配置、调用日志、导出设置和输出文件都保存在用户自己的电脑上。

## 功能特性

- TXT 小说导入与章节拆分，支持常见中文网文章节格式、扩展章节单位，以及带约束的数字标题兜底识别；有章节时按 30 章一批处理，无法识别章节时按长文本批次处理。
- 批次级 AI 分析，提取人物、关系、地点、伏笔、术语、原文姓名、称谓和性别信息，维护改写所需的一致性资产。
- 姓名映射与角色一致性维护，支持主角改写后姓名强制指定、额外需要女性化的人名、跨分片和跨批次复用姓名映射。
- 批次级 AI 改写，将原男女性别叙事改写为双女主 / 百合文本，并重点处理主角姓名、标题、正文、代词、称谓和男性化描写残留。
- 严谨模式与创意模式：严谨模式更忠于原文，创意模式会更主动强化主角女性身份和双女主互动。
- 分析与改写支持并发分片，可选择 10、6、3 或不并发，以平衡处理速度、失败概率、一致性和 token 消耗。
- 任务预估面板会显示全文规模、当前批次规模、预计请求数、预计等待时间、历史调用成功率、平均耗时和输入输出字符统计，并支持折叠收起。
- 一键分析改写全文与一键分析改写当前批次，支持进度展示、暂停、继续和终止；全文流程会保留每批输出并在结束后导出完整改写稿。
- 可选双专家审查，改写模型完成初稿后，可由第二个审查模型对照原文逻辑、女性化要求、称谓、角色性别和设定一致性进行判定；不通过时会打回改写模型重写，再由审查模型复判。
- 多模型配置，支持 OpenAI 兼容接口和 Gemini；可根据 Base URL 提供常见服务商模型下拉建议，并支持思考模式控制和分项模型诊断。
- 诊断结果会显示在工作台顶部，可手动关闭；顶部还提供 GitHub 地址和检查更新入口。
- 首次打开会显示快速上手弹窗，顶部“帮助”按钮可随时重新打开；工作台中的模型配置、章节和一致性资产区域都支持独立滚动，适配较小窗口。
- 原文 / 改写稿对比、AI 调用日志、输入输出字符数、调用耗时、是否复检、是否思考模式等统计，以及只导出已完成改写章节的 TXT。

## 快速上手

1. 点击“导入 TXT”选择小说文件，软件会自动识别章节并分批。
2. 在“模型配置”填写 Base URL、模型名和 API Key，保存后点击“诊断模型”确认连接、JSON 输出和思考模式兼容性。
3. 进入“设定”，填写主角姓名、改写后姓名（可选）、身材、体型、改写模式和额外要求。
4. 如需双专家流程，可在“设置”里开启改写复检，并选择一个审查专家模型；留空时会使用当前改写模型审查。
5. 回到工作台，先看“任务预估”确认请求量和预计等待时间，再点击“一键分析改写当前批次”或“一键分析改写”。
6. 改写完成后进入“对比”检查原文和改写稿，并导出已完成章节的 TXT。

如果用户在未填写必要设定时点击“分析”“改写”或一键流程，应用会提示先完成设定。没有导入小说时，小说设定不可用。

## 模型配置

Yuri Rewrite 支持 OpenAI 兼容接口和 Gemini。常见 Base URL 示例：

- OpenAI: `https://api.openai.com/v1`
- DeepSeek: `https://api.deepseek.com/v1`
- Kimi / Moonshot: `https://api.moonshot.cn/v1`
- MiniMax: `https://api.minimax.chat/v1`
- Xiaomi MiMo: 以小米开放平台提供的 OpenAI 兼容地址为准
- SiliconFlow: `https://api.siliconflow.cn/v1`

模型名需要填写供应商当前提供的具体模型 ID。应用会根据 Base URL 或模型名识别 DeepSeek、OpenAI、Kimi、MiniMax、Xiaomi MiMo、SiliconFlow、Claude 等常见服务商，并在模型名输入框旁提供可选模型列表。

不同供应商对思考模式、JSON 输出和并发请求的支持不同。思考模式建议保持“自动”；如果诊断提示不兼容，按提示改为“关闭”或保持“自动”，也可以降低并发数或更换模型。

“诊断模型”会发起少量轻量请求，分别检查 API Key / Base URL / 模型名、普通聊天响应、JSON 输出能力和思考模式兼容性。诊断结果会在工作台顶部显示，不需要打开日志也能看到失败原因和建议。

## 双专家审查

“设置”中的“改写复检”开启后，改写流程会从单模型改写升级为双专家决策：

1. 改写专家使用当前工作台选择的模型，负责分析和生成初稿。
2. 审查专家使用“设置”中选择的审查模型；如果未选择，则回退使用当前改写模型。选择独立审查模型时，该模型也需要已经保存可用的 API Key。
3. 审查专家只输出 JSON 判定和问题清单，不直接改写正文。
4. 审查重点包括：是否保留原文事件逻辑、因果、战力和伏笔；主角及指定角色是否完成女性化；是否仍有男性姓名、代词、称谓、身份、身体特征和社会角色残留；未指定角色是否被误改性别；外貌、称谓、关系推进和章节边界是否一致。
5. 如果审查通过，初稿直接进入保存。
6. 如果审查不通过，问题清单会打回给改写专家，由改写专家重新输出当前分片的完整章节稿。
7. 重写稿会再交给审查专家复判；复判仍不通过时，该批次会失败并在日志中保留审查问题，避免保存明显不合格的章节。

双专家审查会显著增加请求数和等待时间。开启后，每个分片最多可能经历“分析、初稿改写、审查判定、打回重写、审查复判”五次模型请求。建议为审查专家选择逻辑能力强、JSON 输出稳定、对长文本一致性检查更可靠的模型。

## 任务预估

任务预估会根据当前小说、批次、并发数、复检开关和最近 AI 日志计算：

- 全文章节数、总字数和批次数。
- 当前批次章节数和字数。
- 当前批次与全文一键流程的预计请求数；开启双专家审查后，会按每分片最多五次请求估算。
- 按并发阶段估算的预计等待时间，而不是把所有分片耗时简单相加。
- 最近调用成功 / 失败次数、平均调用耗时、平均输入字符数和平均输出字符数。

如果当前模型没有足够历史成功日志，预计等待和历史字符会显示“暂无历史数据”，避免给出虚假耗时。

## 本地数据与隐私

- 应用数据存放在当前 Windows 用户的应用数据目录中。
- 替换程序目录不会删除已导入小说、配置和分析数据。
- API Key 会优先存储到 Windows 安全存储；不可用时回退到本地数据库。
- 审查专家模型选择、导出目录、复检开关和并发数属于应用设置，会保存在本地。
- AI 日志会保存模型输出、思考内容、原始响应和耗时统计，但不应包含 API Key 或 Authorization 信息。
- 删除小说时，会同步删除该小说相关的章节、批次、设定、一致性资产、任务、日志和改写数据。
